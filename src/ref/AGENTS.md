# AGENTS.md

## Scope

This file applies to work under `it/`.

## Inferno scene breakdown policy

- Write the scene summary to `inferno.md`.
- Process and write cantos one at a time; do not batch many cantos into a single pass when extending the file.
- Use section headings in the form `## Canto N: xxx`.
- Under each canto heading, use a Markdown table with the columns `行`, `場面名`, and `内容`.
- Divide each canto by natural scene changes in the text, not by an arbitrary fixed interval.
- Plan those scene breaks from the start so each scene stays within about 30 lines when possible; if a draft scene would run longer, look for an earlier natural breakpoint instead of writing a long block first.
- Do not force every scene toward 30 lines when the text does not offer a natural break.
- When the user requests a specific breakpoint or a renamed scene, prefer that instruction over the default grouping.
- Scene names should be short, concrete labels that describe the dominant event or topic of the passage.
- `内容` should be a concise one-sentence summary of what happens in that range.
